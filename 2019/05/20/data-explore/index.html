Recap of Parts 1-4 Data Wrangling

  1. Part 1 Getting Data : I began this data parsing project by using web-scraping techniques to tackle a challenging data scraping task. I used BeautifulSoup and Regular Expressions to scrape and gather various tables of CMBS loan data from an HTML file downloaded from this SEC webpage.
  2. Part 2 Parsing the Headers: In order to combine multiple header rows, spread across multiple CSV files, I had to load each header file, account for encoding differences between Python and the data source, account for duplicate header values, join a hierarchal index of header rows into a single header row, save and concatenate the results.
  3. Part 3 Join and Stack: The table data was joined and stacked horizontally and vertically to piece together the puzzle of data tables first explained in Part 1: Getting Data. Finally, the parsed header row was stacked on to the table data.
  4. Part 4 More Cleanup: Here, I got rid of annoying symbols and characters in the headers, deleted columns which contained all NA values and deleted duplicate columns (the repeated indices for each table on the web page).

Data Science is 80% Data Preparation

There is an axiom that states that 80% of a Data Scientist's time is spent acquiring, cleaning and preparing data. I believe this Forbes article and the survey cited in the article, done by CrowdFlower, now Figure Eight, is the main source of this propagated rule of thumb. In my experience, the 80% rule is absolutely true if not understated.

It is clear to see that, after 4 blog posts and many lines of code, that acquiring, parsing and joining data can consume the majority of your time when working on a data project (this is certainly the case for this project). Data preparation is loathed by some in the field of Data Science and is sometimes, kind of pejoratively, referred to as "janitorial work" (not that there is anything wrong with janitors. I think humans are decades away from producing a robot that can adequately clean a public restroom). However, I have encountered some profoundly complex and thought provoking challenges when working through the aforementioned "janitorial" data preparation stage (my previous 4 blog posts in particular!) and I feel that I have become a better programmer and data analyst for it.

Basic Dataset Characteristics

There are four important components of datasets that I will cover that I think are immediately "must know" characteristics for any dataset.

  1. The number of columns and rows.
  2. The name of each column.
  3. The datatypes of each column.
  4. The number of null values for each column.
