Getting Data: Parsing HTML Files

Sun 23 December 2018
By Sonny Torres

Why SEC.gov?

After working with Python to manipulate and analyze commercial loan data at my job, I wanted to share my projects and start a data science blog. However, the loan data we used was subject to copyright protection and using it for a public blog was not possible. Luckily, the Securities and Exchange Commission website publishes the same type of commercial loan data. The data used at my job was a monthly updated snapshot of commercial property loan data, which showed the performance of the properties and the mortgage payments being made, and the SEC data reflects a given group of loans that were just originated, before any payments are made.

Understanding the Data

Working with one type of data has made me appreciate the importance of understanding the scope and limitations of a given dataset. For example, does a column of data represent a monthly figure or a cumulative figure? Is the payoff date of a loan really the payoff date or are there exceptions? Before getting into the data, I want to describe at a higher level what the data covers. In this instance, we are looking at CMBS data. Commercial Mortgage-Backed Securities (CMBS) are Bonds backed by a group of commercial mortgages. The bonds are then sold to investors on the open market and thus subject to SEC regulations. The SEC data being used in this post covers one pool, or group, of CMBS loans. The CMBS data points can be thought of as belonging to one of two main categories:

  1. Mortgage data (Loan Balance, Interest Rate, Due Date, etc.)
  2. Commercial property data (Location, Property Type, Square Feet, etc.)

The Goal

My goal with this project was twofold.

  1. To acquire commercial loan data that I could freely use and publish in my data analysis blog/portfolio.
  2. Create a script to achieve 1. but with rules that could generalize with minimal modifications and work on many other CMBS HTML files available on the SEC website.

Let's Get Started

The data set, which represents one pool of mortgage loans, is split into many different tables that are vertically lined up on one page. Because I wanted tabular data, I had to first understand how the various tables were organized in order to put them together in a tabular format. The charts below illustrate how the first four tables are arranged and how they must be transformed in order to fit together as a single dataset like a CSV or excel file. For simplification we can pretend the data set has 10 Commercial Properties.



Step 1: Analyze the layout. We can see four tables with various headers, columns and rows (See Legend). Each column is broken up into two separate tables. So, Properties 1-5 are in Table 1 and Table 2 lists the other 5 properties, Properties 6-10. Similarly, Table 4 is a continuation of Table 3.



Step 2: Tables 1 & 2 need to be glued together and 3 & 4 need to be glued together, both along the horizonal axis.



Step 3: The pairs of tables will be joined together on their respective vertical axes.

Steps 2 & 3, when done iteratively over every 2 tables in the html file, will result in a single tabular data set. The table layout ended up being one of the easier steps in the process. The layout of the data points within the tables required many rules to properly extract.


The HTML Parsing Code

This code was constructed by carefully examining the HTML itself, brushing up on BeautifulSoup4 and RegEx and then creating my main script that completes the task (the very long for loop at the end). To avoid unecessarily repeating verbose instructions in the main script, I pulled out the instructions I used repeatedly and turned them into helper functions that I could then "plug-in" to my script.

But First, import the various libraries and the HTML file itself.

from bs4 import BeautifulSoup
from bs4 import Tag
from collections import defaultdict
import csv
import re
import os


working_dir = r'C:\Users\Username\Desktop\filename'
os.chdir(working_dir)
filename = 'JPC11C05.html'

with open(filename) as sec:
    soup = BeautifulSoup(sec, 'lxml')

We start with a list comprehension that grabs all the data tables by targeting the "table" HTML tag. After studying the HTML layout, you can see the data tables that contain data, and not paragraphs of words, do not have a 'border' attribute in their "table" tag.

tables = [
     table for table in soup.findAll("table") if 'border' not in table.attrs
]

table_count = len(tables)

The Helper Functions

Because the table headers came with their own unique challenges, I wanted to split and keep track of my headers and table data separately. In order to split up the table data from the header data, I first needed to see what distinguished a header from the table data. The table consists of rows that all have the HTML tag <tr> or "table row". Each row has data, or "cells" in excel-speak, and these cells are organized with the <td> or "table data" HTML tag. Within the <td> tag, there is a "font" tag with a "style" attribute. After a few trial and error

def is_header(tag, keyword=r'bold'):
    # returns bs4.element.Tag or None
    return tag.find("font", {"style": re.compile(keyword)})


# Criteria #2.
# Does NOT contain "BORDER-BOTTOM" in the style attribute
def unwanted_header(tag):
    for data in tag.findAll('td'):
        if 'colspan' in data.attrs:
            colspan_val = int(data['colspan'])
            if colspan_val > 2:
                return True
            else:
                return False


# Simple function to see if tag object passes Criteria # 1 & 2.
def is_good_header(tag):
    if is_header(tag) and not unwanted_header(tag):
        return True
    else:
        return False


# Criteria #3, Cells must span the same number of columns.
def cells_same_colspan(colspan_dict):
    # reduce colspan dict values
    max_colspan_per_row = [max(val) for val in colspan_dict.values()]
    if max(max_colspan_per_row) == 1:
        return True
    else:
        return False

# -------------------THE FUN PART---------------------------------------------------------------------------------------
# If criteria #3 is not met, we create a nested dictionary from our simple colspan dictionary.
# The nested_colspan_dict is a record of rows and all of their colspan values in each row.
# This object will be used to store all the information needed to make changes to the actual
# cells of data.


def nested_colspan_dict(colspan_dict):
    nested_colspan_obj = {
        key: [colspan_tuple for colspan_tuple in enumerate(value)]
        for key, value in colspan_dict.items()}
    return nested_colspan_obj

# Next, since we may need to extract data from the nested dictionary
# multiple times, it makes sense to create a function.
# This function returns a generator object.
# This is helpful in extracting only the values we need
# and not loading everything into memory if it can be
# avoided.


def colspans_to_fix(nested_colspan):
    for key, value in nested_colspan.items():
        for tupl in value:
            # tuples are (colspan_index, colspan_value)
            if tupl[1] != 1:
                # key = row number
                # tuple = (index_of_cell, colspan_value_for_cell)
                yield {key: tupl}


# A function to write the rows to a file.
def write_rows(current_table, filename, data):
    with open(str(current_table) + filename + '.csv', 'w', newline='') as f:
        writer = csv.writer(f)
        # if row avoids blank rows
        writer.writerows(row for row in data.values() if row)


for i in range(table_count):

    table = tables[i]
    header_rows = defaultdict(dict)
    colspan = {}
    line_count = 0

    for row in table.findAll("tr"):
        colspan[line_count] = [int(td_tag['colspan'])
                               if 'colspan' in td_tag.attrs else 1
                               for td_tag in row.findAll('td')]
        if is_good_header(row) and i % 2 == 0:
            header_rows['headers'][line_count] = [data.font.text for data in row.findAll('td')]
            line_count += 1
        elif is_header(row) is None:
            header_rows['tdata'][line_count] = [data.font.text
                                                if isinstance(data.font, Tag) else ''
                                                for data in row.findAll('td')]
            line_count += 1
        else:
            continue
    if cells_same_colspan(colspan):
        write_rows(current_table=i, filename=' header', data=header_rows['headers'])
        write_rows(current_table=i, filename=' table', data=header_rows['tdata'])
    else:
        nested_cs = nested_colspan_dict(colspan)
        bad_colspans = [cs for cs in colspans_to_fix(nested_cs)]
        headers_only = [h for h in header_rows['headers']]
        while not cells_same_colspan(colspan):
            for colspan_dict in bad_colspans:
                for k, v in colspan_dict.items():
                    row_index = k
                    colspan_ind, colspan_val = v
                    colspan_minus_one = colspan_val - 1
                    index_for_insert = colspan_ind + 1
                    if row_index in headers_only:
                        headers = header_rows['headers'][row_index]
                        for insert in range(colspan_minus_one):
                            headers.insert(index_for_insert, '')
                    else:
                        rows = header_rows['tdata'][row_index]
                        for insert in range(colspan_minus_one):
                            rows.insert(index_for_insert, '')
                colspan[row_index][colspan_ind] -= 1
        else:
            write_rows(current_table=i, filename=' header', data=header_rows['headers'])
            write_rows(current_table=i, filename=' table', data=header_rows['tdata'])
