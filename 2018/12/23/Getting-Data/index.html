Getting Data: Parsing HTML Files

Sun 23 December 2018
By Sonny Torres

Problem: Sharing Projects that use Proprietary Data

After working with Python to manipulate and analyze commercial loan data at my job, I wanted to share my projects and start a data science blog. However, the loan data we used was subject to copyright protection and using proprietary data for a public blog was not possible.


The Workaround: SEC.gov

Luckily, the Securities and Exchange Commission website publishes the same type of commercial loan data I analyzed at work but with one key difference. The data used at my job was a monthly updated snapshot of commercial property loan data, which showed the performance of the properties and the mortgage payments being made, and the SEC data reflects a smaller group of loans that were just originated, before any payments were made.


The Goal

My goal with this project was twofold.

  1. To acquire commercial loan data that I could freely use and publish in my data analysis blog/portfolio.
  2. Create a script to achieve 1, but with rules that could generalize with minimal modifications and work on many other CMBS webpages hosted by SEC.gov.

Understanding the Data: What is CMBS?

Commercial Mortgage-Backed Securities (CMBS) are Bonds backed by a group of commercial mortgages. The bonds are then sold to investors on the open market and thus subject to SEC regulations. The SEC data being used in this post covers one pool, or group, of CMBS loans. The CMBS data points fall into four main categories:

  1. Mortgage data (Loan Balance, Interest Rate, Maturity Date, etc.)
  2. Commercial property data (Location, Property Type, Square Feet, etc.)
  3. Securitization Data (characteristics that describe the Securitized structure such as a given loan's loan balance as a proportion of the entire pool/group of loans).
  4. Performance Data (historical financial performance of the commercial properties).

The data set, which represents one pool of mortgage loans, is split into many different tables that are vertically lined up on one page. Because I wanted tabular data, I had to first understand how the various tables were organized in order to put them together in a tabular format. Tables 1 & 2 need to be glued together and 3 & 4 need to be glued together, both along the horizonal axis. Notice how the excess headers, (the grey-shaded headers in the above figure) have been eliminated.


The pairs of tables will be joined together on their respective vertical axes. Because "Prop1, Prop2..etc." represents not only commercial properties in the pool of mortgages, but also the indices for each table, we will have to go back and delete the duplicate indices that will result from merging the tables in this way.

When iteratively joining every 4 tables in this way, the result will be a single tabular data set, albeit with plenty of clean up work left to do. The tricky table layout actually pales in comparison, for me, to the complexity of parsing the header rows and creating rules to account for blank spaces in the table, which I will attempt to explain below.


Understanding the Data: the HTML Structure

This code was constructed by first carefully examining the HTML structure (I often used the command Ctrl+Shift+I to view the HTML followed by Ctrl+F to find what I was looking for and sometimes writing down the results on notebook paper and then creating) my main script that completes the task (the very long for loop at the end of this blog post). The key to understanding the HTML, for me, was understanding the HTML tag structure, as well as what makes certain kinds of data unique, specifically what HTML tag attributes make a given row of data unique.


The Main Tools Used & General Code Structure

The code utilizes the main Python library as well as BeautifulSoup and Regular Expressions . To avoid unecessarily repeating verbose instructions in the main script, I pulled out the instructions I used repeatedly and turned them into helper functions that I could then plug-in to my main script.

The Headers

Because the table headers came with their own unique challenges, I wanted to split and keep track of my headers and table data separately. In order to split up the table data from the header data, I first needed to see what distinguished the headers in the HTML. The table consists of rows that all have the tag <tr>, for each "table row", this includes headers. Each row has individual data points, in Excel these would be called "cells", and each cell of data is organized by the <td> or the "table data" tag. Embedded in the <td> tag, there is a "font" tag with a "style" attribute. I found that all header cells contained the word "bold" in the attribute value for the "style" attribute (see below).


Rule 1: All Headers are Bold

So I made a rule to identify all rows with the "bold" attribute value to identify headers. Below is a function with two parameters, tag and keyword. The tag parameter will receive an HTML tag to search for the keyword 'bold' within the font's style attribute value. The .find method only returns the first instance of the match it finds, which is OK because a table row either has bold words and is a header or there are no instances of the 'bold' string and the table row is not a header. When the function below does not find the "bold" keyword, it will return a None object, which is important to know as we continue building rules.

def is_header(tag, keyword=r'bold'):
    # returns bs4.element.Tag or None
    return tag.find("font", {"style": re.compile(keyword)})

However, I wanted to exclude some of the header rows. Some of the tables have a hierarchal index structure where one large header row describes a group of columns. See the "Hotel Operating Statistics" header which is extraneous for my purposes of transforming data into tabular format.

The above "unwanted header" is differentiated in the HTML from the "wanted headers" by having a large "colspan" value. "colspan" tells us how many columns a given data point spans across.


Rule 2: Skip All Headers with Colspan Value > 2

I then searched the HTML document manually in my browser, using Ctrl+f, for "colspan". There are only 20 instances of "colspan" in the entire HTML document. I saw that the highest colspan value for a good header was 2. So, any colspan values over 2 will be disregarded and not written to the CSV file. To be clear, in this HTML file, all colspan attribute values are associated with headers but not all headers are associated with a colspan attribute. The function below will take the <tr> tag as the argument for the tag parameter and the colspan_limit default value is 2. colspan_obj test test test

def unwanted_header(tag, colspan_limit=2):
    colspan_obj = tag.find("td", {"colspan": re.compile(r'\d+')})
        if colspan_obj:
            colspan_val = int(colspan_obj['colspan'])
            if colspan_val > colspan_limit:
                return True
            else:
                return False

Then I created a simple function to combine the two previous functions.

def is_good_header(tag):
    if is_header(tag) and not unwanted_header(tag):
        return True
    else:
        return False

But what about the good header data points with a colspan equal to 2? These must be identified to properly align the header rows with the actual data rows. When converting each <td> data point to a CSV file, if a given column header has a colspan of 2, the header cell will still be written as one cell in the csv file (one <td> tag = one cell in excel). In the image below, the figure labeled "HTML" reflects how the website's tags are organized in this situation. Two <td> tags (one yellow, one blue) are underneath a single <td> header value ("Other Reserve Cap") that spans two columns. This makes sense in an HTML context. However, because my script iterates over every <td> tag and writes each data point to a cell in my CSV file, I ran into the cell alignment problem illustrated in the figure below labeled "CSV Result".


Below is a simple boolean function that returns True when all <td> tags span the same number of columns, 1 each, and returns False when we run into the issue shown above. This is achieved by using the max() function on each value in the colspan_dict which will be shown later.

def cells_same_colspan(colspan_dict):
    # reduce colspan dict values
    max_colspan_per_row = [max(val) for val in colspan_dict.values()]
    if max(max_colspan_per_row) == 1:
        return True
    else:
        return False

Infinite Nest

Like the book Infinite Jest, where the author has a tendency to write one thought, which leads to another thought, and another thought, until one sentence practically covers an entire page, the code here ended up being very... nested.

If the function above, cells_same_colspan, returns False, meaning there is a colspan value not equal to one, then we need to create a nested dictionary. The nested_dictionary, nested_colspan_dict, is a record of rows and all of their colspan values in each row. More specifically, This object will be used to store all the information needed to make cell alignment changes to the actual cells of data.

def nested_colspan_dict(colspan_dict):
    nested_colspan_obj = {
        key: [colspan_tuple for colspan_tuple in enumerate(value)]
        for key, value in colspan_dict.items()}
    return nested_colspan_obj

# Next, since we may need to extract data from the nested dictionary
# multiple times, it makes sense to create a function.
# This function returns a generator object.
# This is helpful in extracting only the values we need
# and not loading everything into memory if it can be
# avoided.


def colspans_to_fix(nested_colspan):
    for key, value in nested_colspan.items():
        for tupl in value:
            # tuples are (colspan_index, colspan_value)
            if tupl[1] != 1:
                # key = row number
                # tuple = (index_of_cell, colspan_value_for_cell)
                yield {key: tupl}


# A function to write the rows to a file.
def write_rows(current_table, filename, data):
    with open(str(current_table) + filename + '.csv', 'w', newline='') as f:
        writer = csv.writer(f)
        # if row avoids blank rows
        writer.writerows(row for row in data.values() if row)


for i in range(table_count):

    table = tables[i]
    header_rows = defaultdict(dict)
    colspan = {}
    line_count = 0

    for row in table.findAll("tr"):
        colspan[line_count] = [int(td_tag['colspan'])
                               if 'colspan' in td_tag.attrs else 1
                               for td_tag in row.findAll('td')]
        if is_good_header(row) and i % 2 == 0:
            header_rows['headers'][line_count] = [data.font.text for data in row.findAll('td')]
            line_count += 1
        elif is_header(row) is None:
            header_rows['tdata'][line_count] = [data.font.text
                                                if isinstance(data.font, Tag) else ''
                                                for data in row.findAll('td')]
            line_count += 1
        else:
            continue
    if cells_same_colspan(colspan):
        write_rows(current_table=i, filename=' header', data=header_rows['headers'])
        write_rows(current_table=i, filename=' table', data=header_rows['tdata'])
    else:
        nested_cs = nested_colspan_dict(colspan)
        bad_colspans = [cs for cs in colspans_to_fix(nested_cs)]
        headers_only = [h for h in header_rows['headers']]
        while not cells_same_colspan(colspan):
            for colspan_dict in bad_colspans:
                for k, v in colspan_dict.items():
                    row_index = k
                    colspan_ind, colspan_val = v
                    colspan_minus_one = colspan_val - 1
                    index_for_insert = colspan_ind + 1
                    if row_index in headers_only:
                        headers = header_rows['headers'][row_index]
                        for insert in range(colspan_minus_one):
                            headers.insert(index_for_insert, '')
                    else:
                        rows = header_rows['tdata'][row_index]
                        for insert in range(colspan_minus_one):
                            rows.insert(index_for_insert, '')
                colspan[row_index][colspan_ind] -= 1
        else:
            write_rows(current_table=i, filename=' header', data=header_rows['headers'])
            write_rows(current_table=i, filename=' table', data=header_rows['tdata'])
